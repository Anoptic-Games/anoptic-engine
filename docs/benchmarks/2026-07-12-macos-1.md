# 2026-07-12 macOS run — display-aware sweep, 10 s per point

10s run at several display multiples on my M1 13" Macbook Pro. Short-run GPU columns came up empty — see Notes; sister run is 2026-07-12-macos-2.md.

## System

- Apple M1 (8C, 4P+4E), 16 GB unified, macOS 14.8.7 (23J520)
- Display: 2560x1600 panel, desktop 1440x900 pt @ 2.0x (2880x1800 px backing), 60 Hz
- MoltenVK 1.4.1 over Metal 3, nix vulkan-loader 1.4.341.0

## Runtime environment

- Engine: c9e99f2 (feature-crashreport) plus the working-tree harness/engine rework (ANO_RES/ANO_POS/ANO_FLOAT/ANO_MENU launch knobs; res= realized extent in the [profile] line)
- Build: Release -O3 -DNDEBUG, CMake + Ninja, clang 22.1.8 (nix, arm64-apple-darwin23.6.0)
- Renderer: MoltenVK, mesh shaders off, MSAA 4x, VK_PRESENT_MODE_IMMEDIATE_KHR, unthrottled
- Scene: demo scene, SHADOWMAP, Sponza + viking room + candles, HUD menu open
- Harness: tools/perf/bench_fps_macos.py, default sweep, --dur 10
- Conditions: active user session (Discord renderer + Chrome + WindowServer, roughly 75% of a core combined, contending for the unified GPU)
